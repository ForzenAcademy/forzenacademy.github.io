Class GameObject Primary scene entity in the runtime. `GameObject` is the engine's equivalent of a Unity game object: it owns a transform, optional physics/collision state, arbitrary components, lifecycle hooks, and parent/child hierarchy relationships. Game-specific behavior is usually added by attaching Component instances. Class Component Reusable behavior module that can be attached to a GameObject. Components are the main authoring surface for game logic in the engine. A component receives the same lifecycle hooks as its owning object, and can read or mutate the owner through gameObject after it has been attached. Class GameDriver Browser frame driver for a Game. `GameDriver` connects the game loop to `requestAnimationFrame`, handles fixed timestep catch-up, renders through CanvasRenderer, forwards input state to the global Input singleton, and coordinates audio pausing when the browser backgrounds the page. Variable Assets Global asset manager used by engine helpers and generated game code. Game code usually imports this object directly: ```ts const name = Assets.getString(R.string.player_name); const sprite = Assets.getBitmap(R.drawable.player_idle); ``` Before bootstrap installs a real manager, this is a small dummy provider that throws for most asset types and returns the fallback for localized strings. Class Language Global language bridge for game code. `Language` gives gameplay systems a single way to detect the player's current language across web, editor preview, Android, iOS, and desktop shells. Native hosts can install a LanguageProvider before startup; game code can then call `Language.getLanguage()` without caring where the value came from. Class Prefab Runtime prefab registry. A prefab is a named factory that creates one or more configured GameObject instances. Providers self-register through PrefabProvider, and gameplay code can spawn them by id. Class Vector Static vector math helpers for common gameplay calculations.
